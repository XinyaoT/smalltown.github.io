现象解释：

  • 首先根据报错信息，可以确定嗅探icmp包使用的是原始套接字。

  • 其次从原理角度：原始套接字可以读写内核没有处理的IP数据包，而流套接字只能读取TCP协议的数据，数据报套接字只能读取UDP协议的数据。因此，如果要访问其他协议发送数据必须使用原始套接字，所以我们也可以推断出再scapy库中我们制定了包的类型是icmp，那么在底层实现的时候时使用的原始套接字。

  • 而原始套接字的使用需要高权限，因为他涉及到了网络内核，需要高权限。因此当我们用普通用户身份运行sniffer.py就会执行不成功。

核心函数解释：

if pkt.haslayer(ICMP) and pkt[ICMP].type == 8：表示如果嗅探到的包有icmp层，且icmp的类型为8（echo-request），则执行我们的包构造函数。

**IP(src=pkt[IP].dst, dst=pkt[IP].src)**：ip层构造，收到的包的目标ip作为源ip，源ip作为目标ip。

**ICMP(type=0, id=pkt[ICMP].id, seq=pkt[ICMP].seq)**：icmp层构造，其中类型（type）被设置为 0，表示 Echo Reply。id 和 seq 分别设置为与原始 ICMP Echo Request 包相同的值，以确保回复与请求匹配

**pkt[Raw]**：Raw是一个用于表示原始数据负载的层，这一层主要是为了获取原始 ICMP Echo Request 包的数据负载，并将其包含在构造的 ICMP Echo Reply 包中，如果省略这一步，ICMP Echo Reply 可能会被主机A认为是不完整的或错误的。

现象解释：

  • 面对互联网中不存在的主机，hostA能接收到响应包，因为ping发送的包会经过网关，也就是我们的程序可以嗅探到。

  • 面对局域网中不存在的主机，hostA不能够接收到响应包，因为在局域网中，数据包不会经过网关，也就是我们的程序嗅探不到，也就无法发送reply的包，由于hostA 找不到10.9.0.99的MAC地址，导致包不可达。

  • 面对互联网中存在的主机，hostA ping发送的包会被我们的程序嗅探到，所以我们的程序会发送reply的响应包；同时存活的主机也会返回响应包，就会造成hostA收到的响应包冗余（DUP）

  • 任务理解：该任务要求我们使用C语言的原始套接字编程实现包欺骗，通过原始套接字，我们可以重新定义整个数据包的结构，实现各个字段自定义赋值。

  • Step2：cheksum计算，实现在cheksum.c中：

unsigned short \*w = buf;: 定义一个指向输入缓冲区的16位无符号整数指针。

int nleft = length;: 用于跟踪缓冲区中剩余的字节数。

int sum = 0;: 用于保存累加的校验和。

unsigned short temp = 0;: 用于处理奇数字节的临时变量。

while (nleft > 1) { sum += \*w++; nleft -= 2; }: 遍历缓冲区中的16位字，将它们加到累加器中。

if (nleft == 1) { \*(u_char \*)(&temp) = \*(u_char \*)w; sum += temp; }: 处理奇数字节，将其转换为16位字并加到累加器中。

sum = (sum >> 16) + (sum & 0xffff); sum += (sum >> 16);: 将高16位的进位加到低16位，并将所有的进位加回。

return (unsigned short)(~sum);: 返回校验和的反码。

  • Step3：定义通过原始套接字发送ip数据包的函数：

一共分为四个步骤：

  • sock = socket(AF_INET, SOCK_RAW, IPPROTO_RAW);：通过调用socket函数创建一个原始的IPv4套接字。SOCK_RAW表示这是一个原始套接字，可以用于构建自定义的协议头，套接字接收和发送的数据都是从MAC首部开始的。IPPROTO_RAW表示用户负责构建整个数据包，包括IP头，只能用于发送。

  • setsockopt(sock, IPPROTO_IP, IP_HDRINCL,&enable, sizeof(enable));：使用setsockopt函数设置套接字选项，其中IP_HDRINCL表示用户提供完整的IP头部。&enable是一个整数指针，表明启用这个选项。

  • dest_info.sin_family = AF_INET;dest_info.sin_addr = ip->iph_destip;：提供目标信息，将目标IP地址信息填充到dest_info结构体中。AF_INET表示IPv4地址族。

  • sendto(sock, ip, ntohs(ip->iph_len), 0, (struct sockaddr \*)&dest_info, sizeof(dest_info));close(sock);``：发送ip数据包，并关闭套接字。

  • Question 6. 为什么需要root权限才能运行使用原始套接字的程序？如果没有root权限执行，程序会在哪里失败？

创建原始套接字需要root权限。这是因为原始套接字提供了对网络协议栈的直接访问，而这种直接的访问可能导致潜在的网络安全风险。
